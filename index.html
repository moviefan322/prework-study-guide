HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Meta elements are part of the head, they are not visible to the user but contain important information for the brwoser and searche engines
  • 'charset=xxx' is used to define the character base, our default is 'UTF-8'
  • We define our interface layout with meta, under 'name="viewport" content="width-device-width, initial-scale=1.0'"
  • A title section defines what will appear in the browser

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element

Git

  • git status: checks what branch we are currently working on
  • git checkout -b branch-name: creates a new branch and switches to it
  • To add a file to repo:
    1. Navigate to appropriate folder in terminal
    2. Check branch with 'git status'
    3. Make sure you are in main branch with 'git checkout main'
    4. Sync branches with 'git pull origin main'
    5. Create new branch with 'git checkout -b feature/'file name'
    6. Navigate to desired file location and create file with 'touch "file name.extension"'
    7. Begin coding!
  • To save work to GitHub:
    1. Navigate to appropriate directory in terminal
    2. git add -A
    3. git commit -m "description"
    4. git pull origin main
    5. git push origin feature/"name of file" of branch you want to upload
    6. On Github, navigate to appropriate repo
    7. Match the branches, refresh page
    8. Pull Requests -> Create pull request
    9. Add description of changes
    10. Merge Pull Requests
    11. Close Issue (if applicable)
    12. Update local branch with 'git checkout main'
    13. Pull in changes with 'git pull origin main'

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
